## 📝 Criterios de Evaluación

### 1. Conocimiento (30%) 🧠

#### Nivel Excelente (90-100%)
- Explica diferencias entre dict y otras estructuras
- Comprende hashability y claves válidas
- Domina todos los métodos de diccionarios
- Entiende complejidad O(1) de operaciones

#### Nivel Bueno (70-89%)
- Crea diccionarios correctamente
- Usa métodos principales sin errores
- Comprende iteración sobre diccionarios
- Aplica comprehensions básicas

#### Nivel Básico (50-69%)
- Crea diccionarios simples
- Accede a valores por clave
- Itera con bucle for básico
- Necesita ayuda con métodos avanzados

#### Nivel Insuficiente (<50%)
- Confunde diccionarios con listas
- No comprende concepto clave-valor
- Errores frecuentes en sintaxis
- No completa ejercicios

## 📋 Lista de Verificación

### Diccionarios Básicos
- [ ] Crear diccionarios vacíos y con datos
- [ ] Acceder a valores con `[]` y `get()`
- [ ] Agregar y modificar pares clave-valor
- [ ] Eliminar con `del`, `pop()`, `popitem()`
- [ ] Verificar existencia con `in`
- [ ] Obtener longitud con `len()`

### Métodos de Diccionarios
- [ ] Usar `keys()` para obtener claves
- [ ] Usar `values()` para obtener valores
- [ ] Usar `items()` para obtener pares
- [ ] Aplicar `update()` para combinar
- [ ] Usar `setdefault()` para valores por defecto
- [ ] Aplicar `copy()` vs asignación directa

### Iteración
- [ ] Iterar sobre claves (forma por defecto)
- [ ] Iterar sobre valores con `values()`
- [ ] Iterar sobre items con `items()`
- [ ] Usar unpacking en iteración
- [ ] Combinar con `enumerate()`

### Dict Comprehensions
- [ ] Crear diccionarios con comprehensions
- [ ] Aplicar condiciones (filtrado)
- [ ] Transformar claves y/o valores
- [ ] Invertir diccionarios
- [ ] Comprehensions anidadas

### Diccionarios Anidados
- [ ] Crear estructuras multinivel
- [ ] Acceder con encadenamiento de claves
- [ ] Modificar valores profundos
- [ ] Iterar recursivamente
- [ ] Aplanar estructuras anidadas

## 💡 Consejos para Éxito

1. **Practica CRUD**: Create, Read, Update, Delete son la base
2. **Usa get()**: Evita KeyError en producción
3. **Comprende hashability**: Solo inmutables como claves
4. **Domina items()**: La forma más útil de iterar
5. **Prueba edge cases**: Diccionarios vacíos, claves inexistentes
